修改Win10终端编码为UTF8

发表于 2021-02-27 分类于 学习总结

修改终端编码为UTF8

说在前面的话

文章起因：由于 GBK 编码的局限性，本人在编程时总是会使用 utf8 编码，在不使用集成开发环境的时候，编写好的代码在 win10 终端输出时，会出现中文乱码，虽然使用英文便可最有效的解决问题，但是不方便和别人进行合作。所以便想修改电脑编码，但是修改电脑编码会出现大量未知问题，比如曾经用 GBK 编码编写的 word 、txt 等文档都会乱码。所以又想到了只修改终端编码。

一、临时性修改，只对本次有效

该方法对 cmd 和 powershell 都有效，此处演示 cmd

  • 首先打开 cmd（或者powershell），在左上角右击选择属性（properties），在打开的面板中可以看到默认编码为 GBK

  • 在终端输入命令 chcp 即可查看当前编码的编号（GBK：936，UTF8：65001）

  • 输入命令 chcp 65001 修改终端编码（仅本次开启有效，这种方式对powershell 也有用）

二、永久修改 cmd 编码

  • 通过修改注册表永久修改终端编码，打开注册表 win + r --> regedit

  • 找到Command Processor

  • 没有 autorun 字符串值则新建字符串值，存在则直接修改，输入数值数据chcp 65001，点击OK（确定）保存，即可完成设置了

  • 打开注册表，找到如下路径

  • 修改 CodePage 值为 fde9 ，保存即可，我这里已经修改了，没修改的应该是 3a8 ，附一张win10 所有编码表

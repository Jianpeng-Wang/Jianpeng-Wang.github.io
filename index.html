Scoop命令详解

一、总述

scoop的命令都是以以下格式呈现的

其中，<command> 部分如下：

alias 管理scoop的别名，比如你可以自己定一个scoop命令，用来代替你觉得不好敲或不好记的命令
bucket 管理Scoop的bucket，包括添加，查看，删除等
cache 显示或者清除下载缓存，有时因网络问题导致安装多次都未安装成功，可以尝试清除下载缓存，重新安装
checkup 检查潜在的问题
cleanup 清除旧版本的应用，scoop在更新软件后不会删除旧版本，需要自己手动删除
config 设置或查看scoop配置
create 创建一个自定义应用程序清单
depends 列出应用的所有依赖信息
export 将所有已安装的应用导出到一个文件
help 显示命令帮助
hold 禁止应用更新
home 打开 APP manifest.json 标注的网站主页
info 显示应用信息
install 安装应用
list 列出已安装的应用
prefix 返回指定应用程序的路径
reset 重置应用程序以解决冲突，可以用来jdk切换版本
search 在以添加的bucket中查找应用
status 检查应用更新
unhold 允许应用更新，和hold相对
uninstall 卸载应用
update 更新scoop或更新应用
virustotal 在virustotal.com上寻找应用程序的 hash 散列
which 返回可执行文件的位置

寻找更多 bucket

3、chche

scoop 有 cache 这一功能，在安装位置也能找到相应目录，它保存了你的下载记录，可以理解为保存了你的软件安装包，这可一使你在删除某个APP后重新安装时不用在下载安装文件迅速安装，当然问题在于时间长了，你的这些数据将是很占空间的，所以在存储空间很紧张的时候需要手动删除。

该命令用来查看和删除缓存（指定APP或全部）

  • 用法
  • show 使用格式
  • rm 使用格式

4、checkup

该命令会进行一系列测试，用来检查可能发生问题的项，需要管理员身份，按提示操作即可。

  • 用法

5、cleanup

scoop 的一个问题就是安装新版本后，旧的版本不会自动删除，需要手动删除。

该命令用来清理软件的旧版本

  • 用法

6、config

scoop 的配置文件位置： C:\Users\xxxx\.config\scoop\config.json

该命令用来查看、设置、删除 scoop 配置。

  • 用法
  • 查看配置项
  • 设置配置项
  • 移除配置项
  • 常见配置

7、create

如果你自己开发了一个软件（或者没有在 scoop 中存在的第三方软件），想用 scoop 进行管理，可以使用这个命令创建一个 manifest.json 对你的应用进行描述。

  • 用法

8、depends

该命令用来查看应用的依赖。比如安装 Python 必须先安装 dark 和 lessmsi（安装Python时会自动安装，不必担心），你可以使用该命令查看。

9、export

该命令可以将已安装的 APP 列表导出成一份文档，默认导出到当前所在位置。

  • 用法

10、help

该命令用来查看 scoop 的命令帮助

  • 用法

11、hold

该命令用来禁止应用的更新，即当你使用scoop update <app> 时会提示已经禁止更新。

这应用于一些不适合频繁更新的软件，如：MySQL

  • 用法

12、home

该命令用来打开应用的网站主页，一般是官网。定义在对应 APP 的 manifest.json 文件中。

  • 用法

13、info

查看应用信息，即将对应 APP 的 manifest.json 文件内容打印在屏幕上。

  • 用法

14、install

该命令用于安装应用。

  • 用法

15、list

该命令可以列出已经安装的应用，支持正则匹配。

  • 用法

16、prefix

返回应用的安装位置。

  • 用法

17、reset

解决特定应用冲突，比如切换版本。假如你同时安装了最新版本的 Python 和 Python27，那你就可以使用这个命令来切换版本。

  • 用法

18、search

该命令用来搜索应用（在所有bucket中搜索，无论是否已经安装）。同样支持正则匹配（同 list 命令的使用，此处不赘述）。

  • 用法

19、status

该命令用来显示所有可更新的软件。已经被禁止更新的应用也会显示，但是不会更新。

  • 用法

20、unhold

该命令用来解除禁止应用更新，与 hold 命令相反。

  • 用法

21、uninstall

该命令用来卸载应用。

  • 用法

22、update

该命令用于更新 scoop 或应用。与 install 命令相差无几，不再赘述。

  • 用法

23、virustotal

该命令可以用来查看应用的 hash 散列或者检验文件安全性。不常用，且比较麻烦，这里不做解释。想研究的小伙伴可以自行研究。在 Powershell 里输入 scoop help virustotal 即可查看相关信息。

24、which

该命令用于查看应用的可执行文件位置。

  • 用法

换机刷机之Scoop恢复

发表于 2021-02-14 分类于 开发工具

如果你不想再一个一个的添加bucket，一个一个的搜索应用，一个一个的敲命令下载，并且不想丢失原来的数据，那么，这篇文章应该对你有用。

以下命令推荐使用Powershell执行

第一步、将你原来的scoop安装文件夹复制到新电脑或新系统上，位置尽量一样（不一样的没试过，我猜可以）

第二步、将你原来的scoop环境变量（SCOOP和SCOOP_GLOBAL）搬过来（如果你的新位置和旧位置不一样了，那你的变量值也要做相应改变），并且在环境变量path中添加 你scoop安装路径\shims （两个路径要对应，普通安装的添加在用户环境变量，全局安装的添加在系统环境变量）

第三步、以管理员身份打开PowerShell，运行 Set-ExecutionPolicy RemoteSigned -scope CurrentUser 后输入 Invoke-Expression (New-Object System.Net.WebClient).DownloadString('https://get.scoop.sh') 及正常的scoop安装步骤。

第四步、这时会报错，scoop以安装，然后退出Powershell 重新进入，输入命令 scoop reset * 就可以恢复了
